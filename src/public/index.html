What is a Self-Hosted Runner?

A self-hosted runner is a machine you manage yourself that runs GitHub Actions workflow jobs. Unlike GitHub-hosted runners (which spin up a fresh VM per job on GitHub's infrastructure), a self-hosted runner is always-on, runs on your own hardware or cloud instance, and gives you full control over the environment — installed software, network access, IAM roles, disk size, and cost.

☁️

GitHub-Hosted Runner

Fresh VM per job. Managed by GitHub. Limited to 6h jobs. Costs minutes from your quota. No persistent state.

🖥️

Self-Hosted Runner

Your own machine. Persistent state between jobs. Access to private VPCs. Free compute (you pay AWS). Full control.

How to Run This on AWS

Prerequisites

  • AWS account with an EC2 key pair
  • GitHub repository (this one: ismailrz/github-actions-self-hosted-runner-cicd)
  • EC2 instance — Ubuntu 22.04+, t3.micro or larger, 20GB root volume
  • Security group: outbound 443 open (runner phones home to GitHub)
  1. Launch an EC2 instance Ubuntu 22.04 AMI · t3.micro · 20GB gp3 EBS volume · assign a security group with outbound HTTPS.
  2. Copy the setup script to EC2 
    scp -i ~/your-key.pem scripts/setup-ec2-runner.sh \
    ubuntu@<EC2-IP>:~/
  3. Get a registration token from GitHub Go to Repo → Settings → Actions → Runners → New self-hosted runner and copy the --token value.
  4. SSH in and run the setup script 
    ssh -i ~/your-key.pem ubuntu@<EC2-IP>

chmod +x ~/setup-ec2-runner.sh
~/setup-ec2-runner.sh \
  --repo   https://github.com/ismailrz/github-actions-self-hosted-runner-cicd \
  --token  <YOUR-TOKEN> \
  --name   ec2-runner-prod \
  --labels self-hosted,linux,x64,ec2
    The script installs Docker, AWS CLI, downloads the runner binary, registers it with GitHub, and starts it as a systemd service.
  5. Confirm the runner is online Go to Settings → Actions → Runners — you should see Idle next to your runner.
  6. Push code and watch CI run Any push to main or develop triggers the CI workflow on your EC2 runner.

Managing the Runner

# Check status
sudo systemctl status actions.runner.*.service

# Live logs
sudo journalctl -u actions.runner.*.service -f

# Restart
sudo systemctl restart actions.runner.*.service

# Remove runner
cd /opt/github-runner
sudo ./svc.sh stop && sudo ./svc.sh uninstall
./config.sh remove --token <REMOVE_TOKEN>
